# llm_brain Agent 核心需求规格说明书 (PRD)

## 1. 引言
本项目旨在将当前基础的 `llm_brain` 框架演进为具备认知、记忆、学习与自我演化能力的高阶大模型 Agent。核心围绕对多模态输入/输出的深层次特征提取、记忆分类持久化、预测与反思机制，以及记忆-技能的转化能力构建。

## 2. 核心架构与功能需求

### 2.1 认知与特征工程系统 (Cognitive & Feature System)
为解决大模型对庞杂上下文的钝感，建立精简的特征维度体系：
*   **多维特征提取**：需调用 LLM 针对输入信息进行无前置上下文的独立关键字提取。提取时可附加“领域标签”作为先验提示。特征上限受系统约束：**子任务的特征关键字控制在 3 到 5 个，总任务提取上限不超过 30 个**。
*   **一句话缩写/描述机制**：除了关键字提取外，必须附加通过 LLM 进行“一句话压缩描述”的功能。该描述向量在逻辑层级上与关键字特征等同，用于提升后续模糊匹配与语义感知的敏感度。
*   **领域 (Domain) / 空间标签化**：系统**内置一套基础（Base）领域树**。对话或任务结束后，需调用 LLM 参考内置分类模型对全局内容进行复盘，提炼所属“领域（标签）”，避免生成散乱的近义词。该维度用于实现知识网的子空间隔离与聚类（同领域的数据被划分到共同区块中）。

### 2.2 持久化记忆与存储系统 (Memory & Storage Strategy)
要求最大粒度地保留输入输出的数据，并与特征做耦合。
*   **全量信息流记录**：存储基础结构需包含：时间戳、消息ID/对话共有ID、领域标签、特征关键字组、一句话描述、原始输入/输出内容。
*   **阶梯式信息存储策略**：
    1.  **轻量数据**：直接入库/文本化保存。
    2.  **重度数据**：采取独立文件备份。
    3.  **超大数据**：记录文件绝对路径，需调用时进行运行时加载。
*   **两段式信息读取链路 (特征匹配原则)**：逻辑推演和任务评估时，必须优先仅加载“特征关键字”与“一句话描述”参与决策；当该摘要层信息量不足以支撑大模型输出可靠判断时，再触发加载原始信息的完整数据流。
*   **重复加强机制**：摒弃复杂的生命周期遗忘算法。统一通过在复用时的“权重+1”逻辑来实现重要度的区分。即直接对近期数据以及历史重复提及内容在被调用或提取时自动增加 1 点权重。

### 2.3 预测、重现与反思体系 (Prediction & Reflection System)
*   **重现能力 (Replay)**：需具备加载历史全链路数据重新推演的能力。在重现时，能向原数据流中注入独立特征（如：打上`重现`、`反推思考`、`假设性虚拟`等标签）进行逻辑沙盒测试。
*   **预测与闭环验证**：
    *   处理事件需产出初步的“预测结果”。
    *   在事件发展后自动对比验证“预测结果”与“实际结果”。
    *   **纠偏分支与异常阻断**：若验证失败，需触发自主反思工作流（重新评估：是否特征提取不足？缺乏外部信息？推演逻辑错乱？）。如果依然不可解决，**先中断并主动询问 User 以获取额外支持**。在判定用户告知确实不可解决后，再进行记录弃置和归档。
    *   若预测并验证稳定且高频成功，则该固定逻辑处理输入输出映射应能**转化为固化的技能**。

### 2.4 技能树与任务调度 (Skill & Task Management)
*   **记忆-技能的转化互通**：将 MD 文档（含工作流描述、CLI 指南、附件脚本等）作为技能的标准载体。需提供一个核心的“记忆转化为技能”的方法通道，以及通过解析外部技能包转变为 Agent “先验记忆”的通道。
*   **Markdown技能结构设计**：必须采用明确且严格的 Front-matter (YAML) 用于约束 LLM 读取。技能开头基础模板如下：
    ```yaml
    ---
    name: "技能名称"
    confidence: 50
    keywords: ["特征关键字1", "特征关键字2", "特征关键字3"]
    description: "该技能的一句话核心描述..."
    entry_node: "主调用入口"
    ---
    ```
*   **任务拆解规范**：所有复杂任务请求需要被分割为细粒度的“子任务”。
    *   每个子任务的上下文流向最多只能挂载/调用 **1 个技能**。
    *   父级（完整）交流任务必须囊括、组装其下属所有子任务的特征关键字。
*   **可信度溯源与评分系统 (Confidence Scoring)**：所有抽取的常规记忆及技能必须引入 `可信度/概率` 权重。
    *   **硬性规则指引**：开发人员预设或用户手动挂载的基础技能、核心文档，初始可信度 `>= 95%`。
    *   **模型涌现关联**：由 LLM 自行归纳记忆推演或内部反思总结生成的动态技能，初始可信度 `< 50%`，随预测验证成功率逐渐拔高。
